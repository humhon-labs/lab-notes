# Agent Skills — 에이전트에게 지식과 워크플로를 담아 주는 열린 포맷

> 🧩 **Agent Skills**는 AI 에이전트에게 전문 지식과 반복 가능한 워크플로를 부여하는 **경량 오픈 포맷**입니다. 스킬은 `SKILL.md` 파일 하나를 담은 폴더이며, 필요할 때만 지침을 불러오는 **점진적 공개(progressive disclosure)** 방식으로 동작합니다. Anthropic이 만들어 오픈 표준으로 공개했고, Claude Code·Cursor·Gemini CLI·GitHub Copilot 등 여러 에이전트가 채택했습니다. 이 노트는 그 개념과 구조를 정리합니다.

## 1. Agent Skills란 무엇인가

에이전트(agent)는 점점 똑똑해지지만, 실제 업무를 안정적으로 해내는 데 필요한 **맥락(context)** 을 대개 갖고 있지 않습니다. "우리 회사는 계약서를 이렇게 검토한다", "이 데이터 파이프라인은 이 순서로 돌린다" 같은 절차적 지식은 모델 가중치 안에 들어 있지 않기 때문입니다.

Agent Skills는 이런 **절차적 지식과 조직·팀·개인 고유의 맥락**을 이식 가능한(portable), 버전 관리되는 폴더로 묶어, 에이전트가 **필요할 때 불러오도록(load on demand)** 하는 포맷입니다.

- **핵심 정의**: 스킬(skill) = `SKILL.md` 파일을 담은 **폴더**. 이 파일은 메타데이터와 지침으로 이루어지며, 필요하면 스크립트·참조 문서·템플릿 같은 리소스를 함께 번들할 수 있습니다.
- **성격**: 특정 벤더에 묶이지 않은 **열린 포맷**. 한 번 만든 스킬을 여러 호환 에이전트에서 재사용할 수 있습니다.

## 2. 왜 필요한가

> 📦 스킬은 "프롬프트에 매번 붙여 넣던 노하우"를 **재사용 가능한 파일**로 외부화합니다.

스킬이 에이전트에게 주는 것:

- **도메인 전문성**: 법무 검토 절차, 데이터 분석 파이프라인, 발표자료 포맷 규칙 같은 특수 지식을 재사용 가능한 지침·리소스로 포착합니다.
- **반복 가능한 워크플로**: 여러 단계로 이뤄진 작업을 일관되고 감사(audit) 가능한 절차로 만듭니다. 매번 결과가 들쭉날쭉하지 않습니다.
- **제품 간 재사용**: 한 번 만들면 스킬을 지원하는 어떤 에이전트에서든 그대로 씁니다. 도구를 바꿔도 노하우가 따라옵니다.

## 3. 스킬의 구조

스킬은 최소한 `SKILL.md` 하나를 담은 디렉터리입니다. 나머지는 모두 선택입니다.

```text
skill-name/
├── SKILL.md          # 필수: 메타데이터 + 지침
├── scripts/          # 선택: 실행 가능한 코드
├── references/       # 선택: 추가 문서(필요 시 로드)
├── assets/           # 선택: 템플릿·리소스
└── ...               # 그 밖의 파일/디렉터리
```

- `scripts/` — 에이전트가 **실행**할 수 있는 코드. 자체 완결적이거나 의존성을 명시하고, 친절한 에러 메시지와 예외 처리를 갖추는 것이 권장됩니다. (Python·Bash·JavaScript 등, 지원 언어는 에이전트 구현에 따라 다름)
- `references/` — 필요할 때만 읽어 들이는 **참조 문서**. 긴 내용을 `SKILL.md`에서 분리해 두는 곳입니다.
- `assets/` — 템플릿, 양식 등 재사용 **리소스**.

## 4. `SKILL.md` 포맷

`SKILL.md`는 **YAML 프론트매터 + 마크다운 본문** 구조입니다. 프론트매터는 에이전트가 이 스킬을 언제 쓸지 판단하는 최소 정보를 담고, 본문은 실제 지침을 담습니다.

```markdown
---
name: pdf-processing
description: PDF에서 텍스트·표를 추출하고, 양식을 채우고, 여러 PDF를 병합한다.
  사용자가 PDF·양식·문서 추출을 언급하거나 PDF 문서를 다룰 때 사용한다.
license: Apache-2.0
metadata:
  version: "1.0"
---

# PDF 처리

## 단계별 지침
1. ...

## 입력/출력 예시
...

## 흔한 예외 상황
...
```

프론트매터 필드:

| 필드 | 필수 | 제약 |
|------|------|------|
| `name` | ✅ | 최대 64자, 소문자·숫자·하이픈만. 하이픈으로 시작/끝 불가, 연속 하이픈 불가, **부모 디렉터리명과 일치**해야 함 |
| `description` | ✅ | 최대 1024자, 비어 있으면 안 됨. **무엇을 하는지 + 언제 쓰는지**를 키워드와 함께 기술 |
| `license` | ❌ | 라이선스 이름 또는 번들된 라이선스 파일 참조 |
| `compatibility` | ❌ | 최대 500자. 대상 제품·필요 패키지·네트워크 접근 등 환경 요구사항 |
| `metadata` | ❌ | 임의의 key-value (버전·작성자 등) |
| `allowed-tools` | ❌ | 사전 승인 도구 목록(공백 구분). 실험적 |

> ⚠️ `description`이 스킬의 품질을 좌우합니다. "PDF를 도와줍니다" 같은 모호한 설명 ✕ → "PDF에서 텍스트·표 추출, 양식 채우기, 병합. PDF·양식·문서 추출을 다룰 때 사용" 처럼 **행동 + 트리거 키워드**를 담은 설명 ○. 에이전트는 이 문장만 보고 스킬을 고르기 때문입니다.

## 5. 어떻게 동작하나 — 점진적 공개 3단계

> 🔎 핵심 아이디어: **전체 지침을 항상 메모리에 올려 두지 않는다.** 이름·설명만 상시 유지하고, 실제 지침은 작업이 요구할 때만 로드해 컨텍스트를 아낀다.

1. **Discovery(발견)** — 에이전트는 시작 시 각 스킬의 **`name`과 `description`만** 불러옵니다. "이 스킬이 언제 쓸 만한지" 판단할 최소 정보입니다.
2. **Activation(활성화)** — 작업이 어떤 스킬의 설명과 매칭되면, 그제서야 해당 **`SKILL.md` 전체 지침**을 컨텍스트로 읽어 들입니다.
3. **Execution(실행)** — 지침을 따라 작업을 수행하며, 필요에 따라 번들된 **코드를 실행**하거나 `references/`의 **참조 파일을 로드**합니다.

이 3단계 덕분에 에이전트는 **수많은 스킬을 손에 쥐고도 컨텍스트 부담은 작게** 유지합니다. 전체 지침은 실제로 필요할 때만 펼쳐지기 때문입니다.

## 6. MCP·툴과 무엇이 다른가

<details>
<summary>스킬 vs 다른 확장 방식 — 펼쳐 보기</summary>

- **툴/함수 호출(function calling)**: 에이전트가 호출할 수 있는 개별 능력(API)을 정의합니다. 스킬은 그런 능력을 *언제·어떻게* 조합해 쓰는지의 **절차적 지식**에 가깝습니다. 스킬 안에서 툴을 활용할 수 있습니다.
- **MCP(Model Context Protocol)**: 에이전트를 외부 데이터·도구 서버에 연결하는 프로토콜입니다. 스킬은 서버가 아니라, 파일 폴더로 배포되는 **지침·리소스 번들**이라는 점이 다릅니다. 둘은 상호 보완적입니다.
- **순수 프롬프트**: 프롬프트는 매번 붙여 넣어야 하지만, 스킬은 버전 관리되는 파일로 **한 번 만들어 재사용·공유**합니다.

</details>

## 7. 열린 개발과 채택 현황

- Agent Skills 포맷은 처음에 **Anthropic**이 개발해 **오픈 표준**으로 공개했습니다.
- 이후 다양한 에이전트 제품이 채택했습니다 — **Claude Code, Cursor, Gemini CLI, GitHub Copilot** 등 스킬 호환 클라이언트가 늘고 있습니다.
- 표준은 생태계의 기여에 열려 있으며, GitHub·Discord에서 논의가 진행됩니다. 폴더 하나가 **여러 도구를 넘나드는 공용 지식 단위**가 된다는 점이 핵심 가치입니다.

## 8. 정리 — 실무 관점 요약

- 스킬은 "**프롬프트에 매번 설명하던 것**"을 파일로 굳혀 재사용·버전 관리·공유하게 만드는 장치입니다.
- 잘 쓰인 `description`이 곧 자동 라우팅입니다. 에이전트는 설명만 보고 스킬을 고릅니다.
- 긴 지침은 `references/`로 쪼개고, 반복 실행 로직은 `scripts/`로 빼서 **컨텍스트를 절약**하세요.
- 벤더 중립 포맷이므로, 한 번 만든 스킬 자산은 도구를 바꿔도 살아남습니다.
